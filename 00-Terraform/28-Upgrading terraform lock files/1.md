# Upgrading Terraform Lock Files with Azure Remote Backend

## Prerequisites

Your Azure identity must have the **Storage Blob Data Contributor** role on the storage account used for the backend.

## Steps

### 1. Sign in to the Relevant Repository

Ensure you are authenticated and have access to the correct Azure subscription.

### 2. Run `terraform init` with Backend Reconfiguration

Use the following command to reinitialize Terraform, reconfigure the backend, and upgrade provider lock file entries. Replace the placeholder values (`<>`) with your actual resource details:

```bash
terraform init \
  -reconfigure \
  -backend-config="resource_group_name=<resource_group_name>" \
  -backend-config="storage_account_name=<storage_account_name>" \
  -backend-config="container_name=<container_name>" \
  -backend-config="key=<state_file_key>" \
  -backend-config="subscription_id=<subscription_id>" \
  -backend-config="use_azuread_auth=true" \
  -upgrade
```

The `-upgrade` flag instructs Terraform to check for newer provider versions that satisfy the configured version constraints and update the `.terraform.lock.hcl` file accordingly.
