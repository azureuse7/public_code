# Amazon S3: Simple Storage Service

> S3 is AWS's object storage service — store and retrieve any amount of data at any time. Built for 99.999999999% (11 nines) durability, it is the backbone for backups, static website hosting, data lakes, CDN origins, and application asset storage.

---

## Core Concepts

| Term | Description |
|---|---|
| **Bucket** | Top-level container for objects; globally unique name; tied to a region |
| **Object** | A file + metadata; identified by a key (path-like name) |
| **Key** | The full path to an object (e.g., `images/2024/photo.jpg`) |
| **Prefix** | A key prefix used to organise objects like folders |
| **Versioning** | Keeps multiple versions of an object in the same bucket |

---

## Storage Classes

| Class | Durability | Availability | Min Storage | Use For |
|---|---|---|---|---|
| **Standard** | 11 nines | 99.99% | None | Frequently accessed data |
| **Intelligent-Tiering** | 11 nines | 99.9% | 30 days | Unknown or changing access patterns |
| **Standard-IA** | 11 nines | 99.9% | 30 days | Infrequent access; retrieval fee applies |
| **One Zone-IA** | 11 nines | 99.5% | 30 days | Non-critical infrequent data; single AZ |
| **Glacier Instant Retrieval** | 11 nines | 99.9% | 90 days | Archive with millisecond retrieval |
| **Glacier Flexible Retrieval** | 11 nines | 99.99% | 90 days | Archive; 1–12 hour retrieval |
| **Glacier Deep Archive** | 11 nines | 99.99% | 180 days | Rarely accessed archive; 12-hour retrieval |

## Key Features

### Versioning

Keeps all versions of an object. Protects against accidental deletes and overwrites.

```bash
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled
```

### Lifecycle Policies

Automatically transition objects between storage classes or delete them:

```bash
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration '{
    "Rules": [{
      "ID": "archive-old-logs",
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Transitions": [
        {"Days": 30, "StorageClass": "STANDARD_IA"},
        {"Days": 90, "StorageClass": "GLACIER"}
      ],
      "Expiration": {"Days": 365}
    }]
  }'
```

### Replication

- **CRR** (Cross-Region Replication): disaster recovery, latency reduction
- **SRR** (Same-Region Replication): log aggregation, compliance

### Event Notifications

Trigger Lambda, SQS, or SNS when objects are created, deleted, or restored:

```bash
aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations": [{
      "LambdaFunctionArn": "arn:aws:lambda:us-east-1:123456789012:function:ProcessUpload",
      "Events": ["s3:ObjectCreated:*"]
    }]
  }'
```

## Security

| Mechanism | Description |
|---|---|
| **Block Public Access** | Account/bucket-level setting that overrides all public grants — enable by default |
| **Bucket Policy** | Resource-based JSON policy for cross-account or service access |
| **IAM Policy** | Identity-based policy for fine-grained user/role access |
| **ACLs** | Legacy per-object access control — avoid for new setups |
| **SSE-S3** | Server-side encryption with S3-managed keys (default) |
| **SSE-KMS** | Server-side encryption with AWS KMS keys (audit trail + key control) |
| **SSE-C** | Customer-provided keys |
| **TLS in transit** | Enforce HTTPS via bucket policy |

**Enforce HTTPS via bucket policy:**

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Deny",
    "Principal": "*",
    "Action": "s3:*",
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ],
    "Condition": {
      "Bool": { "aws:SecureTransport": "false" }
    }
  }]
}
```

## Common CLI Operations

```bash
# Create a bucket
aws s3api create-bucket \
  --bucket my-example-bucket \
  --region us-west-2 \
  --create-bucket-configuration LocationConstraint=us-west-2

# Enable versioning
aws s3api put-bucket-versioning \
  --bucket my-example-bucket \
  --versioning-configuration Status=Enabled

# Upload a file
aws s3 cp myfile.txt s3://my-example-bucket/myfile.txt

# Upload a folder (recursive)
aws s3 cp ./dist s3://my-example-bucket/dist/ --recursive

# List objects
aws s3 ls s3://my-example-bucket/

# Download a file
aws s3 cp s3://my-example-bucket/myfile.txt ./myfile.txt

# Sync a local folder to S3
aws s3 sync ./build s3://my-website-bucket --delete

# Generate a pre-signed URL (valid 1 hour)
aws s3 presign s3://my-example-bucket/private.pdf --expires-in 3600

# Delete an object
aws s3 rm s3://my-example-bucket/myfile.txt

# Delete all objects in a prefix
aws s3 rm s3://my-example-bucket/tmp/ --recursive

# Delete bucket (must be empty first)
aws s3api delete-bucket --bucket my-example-bucket --region us-west-2
```

## Static Website Hosting

```bash
# Enable static website hosting
aws s3 website s3://my-website-bucket/ \
  --index-document index.html \
  --error-document error.html

# Make bucket publicly readable (only for public static sites)
aws s3api put-bucket-policy \
  --bucket my-website-bucket \
  --policy '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-website-bucket/*"
    }]
  }'
```

> For production static sites, use **CloudFront + S3 with Origin Access Control (OAC)** — keeps the bucket private and adds HTTPS, caching, and DDoS protection.

## Common Use Cases

| Use Case | Configuration |
|---|---|
| Backups and archives | Versioning + Lifecycle → Glacier |
| Static website | S3 website hosting + CloudFront |
| Data lake | Standard class + Athena + Glue |
| Application uploads | Event notification → Lambda processing |
| Log storage | Lifecycle → Standard-IA → Glacier |
| Secure file sharing | Pre-signed URLs |

---

## Summary

S3 is the most versatile storage service in AWS. It is durable, infinitely scalable, and integrates with virtually every other AWS service. Enable versioning and Block Public Access on every bucket by default, use lifecycle policies to manage cost, and front public content with CloudFront rather than direct S3 access.
