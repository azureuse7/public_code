# EC2: Elastic Compute Cloud

> Amazon EC2 provides scalable virtual servers (instances) in the cloud. You choose the OS, instance type (CPU/RAM), storage, and networking, paying only for what you use. EC2 is the foundation for most compute workloads on AWS.

## Introduction

Amazon Elastic Compute Cloud (Amazon EC2) is a web service provided by AWS that offers resizable compute capacity in the cloud. EC2 lets you launch and manage virtual servers (instances) to run applications on AWS's secure and reliable infrastructure, scaling up or down as needed.

Key highlights:
- Reliable, scalable infrastructure on demand with **99.99% SLA** availability
- Security built into the foundation via the **AWS Nitro System**
- Cost flexibility through Graviton-based instances, Spot instances, and Savings Plans

---

## Key Features

### 1. Elasticity

- **Auto Scaling** — Automatically adjusts instance count to maintain performance under changing load
- **Elastic Load Balancing** — Distributes traffic across multiple instances to improve fault tolerance

### 2. Instance Types

EC2 offers a wide range of instance types optimized for different use cases:

| Family | Optimized For |
|---|---|
| Compute (C) | CPU-intensive workloads |
| Memory (R, X) | In-memory databases, caches |
| Storage (I, D) | High I/O, data warehousing |
| GPU (P, G) | ML training, graphics rendering |
| General (T, M) | Balanced compute/memory |

### 3. Pricing Models

| Model | Description |
|---|---|
| **On-Demand** | Pay by the hour/second, no commitments |
| **Reserved Instances** | Up to 75% discount with 1 or 3-year commitment |
| **Spot Instances** | Bid for unused capacity — best for fault-tolerant workloads |
| **Savings Plans** | Flexible savings in exchange for a consistent $/hour commitment |

### 4. Security and Compliance

- Integrates with **IAM** to control resource access
- Supports encryption of data at rest and in transit
- Compliant with various industry standards and regulations

### 5. Storage Options

| Storage | Type | Description |
|---|---|---|
| **EBS** | Persistent | Block storage volumes that persist beyond instance lifetime |
| **Instance Store** | Ephemeral | Temporary block storage tied to the instance lifetime |
| **EFS** | Shared | Scalable file storage accessible by multiple instances |

### 6. Networking

- **VPC (Virtual Private Cloud)** — Launch resources in an isolated virtual network
- **Elastic IP Addresses** — Static IPv4 addresses for dynamic cloud use
- **Enhanced Networking** — Higher PPS performance, lower latency and jitter

## Launching an EC2 Instance

### Using the AWS Management Console

1. Sign in to the AWS Management Console and navigate to the **EC2 Dashboard**
2. Click **Launch Instance**
3. Choose an **AMI** (OS + software stack)
4. Select an **instance type** based on your CPU/memory needs
5. Configure instance details — network (VPC/subnet), IAM role, etc.
6. Add **EBS storage** and optional tags
7. Configure a **security group** (allow SSH port 22, HTTP port 80, etc.)
8. Review, then select or create a **key pair** for SSH access
9. Click **Launch**

### Using the AWS CLI

Ensure the AWS CLI is installed and configured with your credentials:

```sh
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --count 1 \
  --instance-type t2.micro \
  --key-name MyKeyPair \
  --security-group-ids sg-0123456789abcdef0 \
  --subnet-id subnet-6e7f829e
```

| Parameter | Description |
|---|---|
| `--image-id` | AMI ID to launch from |
| `--count` | Number of instances to launch |
| `--instance-type` | Instance size (e.g., `t2.micro`, `m5.large`) |
| `--key-name` | Key pair name for SSH access |
| `--security-group-ids` | Security group controlling allowed traffic |
| `--subnet-id` | Subnet to place the instance in |
