# Azure Cosmos DB — Azure CLI Guide

## What is Azure Cosmos DB?

**Azure Cosmos DB** is Microsoft's globally distributed, multi-model database service designed to provide high availability, low latency, and scalability for modern applications. It is a **NoSQL** database service that supports multiple data models, including:

- **Document** (using SQL API)
- **Key-Value**
- **Graph** (using Gremlin API)
- **Column-Family** (using Cassandra API)
- **Table** (using Azure Table Storage API)

### Key Features of Azure Cosmos DB

1. **Global Distribution:**
   - Replicate your data across any number of Azure regions.
   - Provides low-latency access to data for users worldwide.
2. **Multiple APIs and Data Models:**
   - Supports SQL, MongoDB, Cassandra, Gremlin, and Table APIs, allowing flexibility in choosing the data model that best fits your application.
3. **Elastic Scalability:**
   - Automatically scales throughput and storage based on application needs.
   - Supports both horizontal and vertical scaling.
4. **Guaranteed Low Latency:**
   - Single-digit millisecond response times at the 99th percentile worldwide.
5. **Comprehensive SLAs:**
   - Offers industry-leading SLAs for availability, latency, throughput, and consistency.
6. **Five Consistency Models:**
   - Provides multiple consistency levels ranging from strong to eventual consistency.
7. **Security and Compliance:**
   - Built-in encryption at rest and in transit.
   - Compliant with various international standards and regulations.

### Use Cases

- **Real-Time Web and Mobile Applications:** Fast access to data across the globe.
- **IoT and Telemetry:** Handling massive amounts of data generated by devices.
- **Gaming:** Managing player data and game state with low latency.
- **Retail and E-commerce:** Ensuring quick access to product catalogs and user information.

## Example: Using Azure CLI to Create and Manage Azure Cosmos DB

Below is a step-by-step example demonstrating how to create an Azure Cosmos DB account, create a database and container, and perform basic operations using the Azure CLI in a Bash environment.

### Prerequisites

- Azure CLI installed and configured
- An active Azure subscription

### Step 1: Log in to Azure

```bash
# Log in to your Azure account
az login
```

This command will open a web browser prompting you to enter your Azure credentials.

### Step 2: Set Variables

Define variables for the resource group, Cosmos DB account, database, and container.

```bash
# Variables
RESOURCE_GROUP="MyResourceGroup"
COSMOS_ACCOUNT="mycosmosdbaccount123"
LOCATION="eastus"
DATABASE_NAME="MyDatabase"
CONTAINER_NAME="MyContainer"
PARTITION_KEY="/myPartitionKey"
```

### Step 3: Create a Resource Group

```bash
az group create --name $RESOURCE_GROUP --location $LOCATION
```

### Step 4: Create an Azure Cosmos DB Account

Create a Cosmos DB account using the SQL (Core) API.

```bash
az cosmosdb create \
    --name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --kind GlobalDocumentDB \
    --locations regionName=$LOCATION failoverPriority=0 isZoneRedundant=False
```

### Step 5: Retrieve the Cosmos DB Account's Primary Key

You will need the primary key to authenticate and interact with the database.

```bash
COSMOS_KEY=$(az cosmosdb keys list \
    --name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --query "primaryMasterKey" \
    --output tsv)

echo $COSMOS_KEY
```

### Step 6: Create a Database

```bash
az cosmosdb sql database create \
    --account-name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --name $DATABASE_NAME
```

### Step 7: Create a Container

A container is a schema-agnostic container of items. Define a partition key for scalability.

```bash
az cosmosdb sql container create \
    --account-name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --database-name $DATABASE_NAME \
    --name $CONTAINER_NAME \
    --partition-key-path $PARTITION_KEY \
    --throughput 400
```

### Step 8: Insert a Document into the Container

First, create a JSON file named `item.json`:

```json
{
    "id": "1",
    "myPartitionKey": "partition1",
    "name": "John Doe",
    "email": "john.doe@example.com"
}
```

Then, insert the document:

```bash
az cosmosdb sql container item create \
    --account-name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --database-name $DATABASE_NAME \
    --container-name $CONTAINER_NAME \
    --item @item.json
```

### Step 9: Query Documents in the Container

Retrieve documents using a SQL query.

```bash
az cosmosdb sql container item query \
    --account-name $COSMOS_ACCOUNT \
    --resource-group $RESOURCE_GROUP \
    --database-name $DATABASE_NAME \
    --container-name $CONTAINER_NAME \
    --query-text "SELECT * FROM c WHERE c.name = 'John Doe'"
```

### Step 10: Clean Up Resources

To avoid incurring charges, delete the resource group, which removes all resources within it.

```bash
az group delete --name $RESOURCE_GROUP --yes --no-wait
```

## Summary

Azure Cosmos DB is a powerful, globally distributed database service that offers multiple data models and APIs to cater to various application needs. Its seamless scalability, low latency, and comprehensive SLAs make it an excellent choice for modern, data-intensive applications.

Using the Azure CLI in a Bash environment, you can efficiently manage your Cosmos DB resources, perform CRUD operations, and automate your database workflows. The example above demonstrates the basic steps to get started with creating and interacting with an Azure Cosmos DB account.

For more advanced scenarios and detailed configuration options, refer to the [official Azure Cosmos DB documentation](https://docs.microsoft.com/en-us/azure/cosmos-db/).
